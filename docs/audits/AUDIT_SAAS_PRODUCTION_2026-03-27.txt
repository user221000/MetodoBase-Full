MetodoBase SaaS Production Audit 2026-03-27
Scope: web app (web/), API (api/), database config (web/database, config), static assets, auth, billing, tests.
Analysts: Arquitecto, Debugger, QA, Implementador (senior review pass)

Executive Summary
- Critical: 3
- High: 6
- Medium: 6
- Low: 2
- Note: One fix applied in this pass (static mount for web assets in api/app.py).

CRITICAL
1) Auth not initialized in api_server path
   Impact: Tokens generated by web auth fail on API; 401/403 and tenant context breaks.
   Evidence: api/app.py uses web.auth_deps without calling web.auth.init_auth().
   Fix: Initialize auth in api/app.py (lifespan or create_app). Optionally lazy-init in verify_token.

2) Database config mismatch in production
   Impact: App fails to boot when DATABASE_URL is missing even if DB_PATH exists; runtime crash.
   Evidence: config/settings.py allows DB_PATH in production; web/database/engine.py ignores DB_PATH.
   Fix: Align settings and engine: either support DB_PATH in engine or disallow it in prod.

3) Web app lacks tenant middleware while DB RLS expects it
   Impact: RLS context not set; cross-tenant risks if RLS is enabled.
   Evidence: web/main_web.py does not add TenantMiddleware; web/database/engine.py tries to set current_tenant.
   Fix: Add TenantMiddleware to web/main_web or guard RLS usage when missing tenant.

HIGH
4) CSRF not enforced if web pages are served from api_server
   Impact: CSRF bypass for HTML pages served via api/app.py.
   Evidence: api/app.py mounts web.pages but does not attach CSRFMiddleware.
   Fix: Serve HTML only from web/main_web.py or add CSRFMiddleware in api/app.py.

5) MercadoPago webhook incomplete
   Impact: Payments not activated; webhook spoofing risk.
   Evidence: api/routes/pagos.py has TODOs for signature/idempotency/activation.
   Fix: Implement verification, idempotency, and subscription activation.

6) Team invite URL duplication
   Impact: Invitation links broken; onboarding fails.
   Evidence: web/routes/team.py builds URL twice for accept-invite.
   Fix: Pass token only or accept full URL without re-concatenation.

7) Hardcoded BASE_URL in email templates
   Impact: Wrong links in staging/custom domains.
   Evidence: web/services/email_service.py uses https://app.metodobase.app.
   Fix: Use settings.BASE_URL to construct URLs.

8) Web/API layer coupling and schema inversion
   Impact: Slows refactor, blocks independent deploys, increases regression risk.
   Evidence: api/app.py imports web.*, web/routes import api.schemas.
   Fix: Extract shared module for auth/models/middlewares and decouple layers.

MEDIUM
9) Plan generation duplicated in multiple layers
   Impact: Behavior drift between API and web; inconsistent outputs.
   Evidence: api/routes/planes.py, web/routes/planes.py, api/services.py.
   Fix: Consolidate into single service.

10) Trial expiration and client limits not enforced
   Impact: Trial never expires, limits ignored.
   Evidence: web/repositories/gym_settings_repository.py returns false or TODO.
   Fix: Implement trial/limit logic or remove unused methods.

11) Migrations not guaranteed in production
   Impact: App fails after deploy if alembic not run.
   Evidence: web/main_web.py skips create_all in prod without a startup check.
   Fix: Add startup check or explicit migration step in deploy pipeline.

12) Rate limiter can block static assets
   Impact: CSS/JS may get 429 under load.
   Evidence: web/middleware/rate_limiter.py excludes docs/health but not /static.
   Fix: Add /static to exclude_paths.

13) Billing uses user_id instead of effective gym_id in some paths
   Impact: Billing applied to wrong entity for team members.
   Evidence: web/routes/billing.py uses usuario['id'].
   Fix: Use effective gym_id from auth context.

LOW
14) Version hardcoded in api/app.py
   Impact: Drift from config.constantes.VERSION.
   Fix: Import VERSION constant.

15) Legacy rate limiter duplicate
   Impact: Confusion and dead code risk.
   Evidence: web/rate_limiter.py unused vs web/middleware/rate_limiter.py.
   Fix: Remove or consolidate.

Testing Gaps (QA)
- Missing HTTP tests for auth endpoints (login/registro/refresh/logout/me)
- Missing end-to-end billing tests and webhook signature tests
- Missing subscription guard tests (enforcement of limits/trials)
- Missing web routes multi-tenant tests (/clientes, /gym/*)
- Missing team invite/accept tests (expired/reused/invalid role)

Databases Reviewed
- SQLite dev: metdobase_web.db (fallback) and auth SQLite (web/auth)
- PostgreSQL prod: DATABASE_URL enforced; RLS context uses current_tenant
- Risk: prod boot mismatch if DB_PATH used without DATABASE_URL

Fix Applied (this pass)
- api/app.py now mounts /static from web/static (fixes missing CSS/skeleton assets)
