SECTION 2 — CRITICAL ISSUES FOUND & FIXED
════════════════════════════════════════════════════════════════════════════════

[FIXED] C-01: Rate Limiter Memory Leak
  File:    web/rate_limiter.py
  Issue:   _cleanup_expired() only removed fully expired buckets, not
           individual stale timestamps within active buckets.
  Fix:     Added trimming of old timestamps in all buckets + safe iteration.

[FIXED] C-02: CSP 'unsafe-inline' for Styles
  File:    web/middleware/security_headers.py
  Issue:   style-src CSP directive used 'unsafe-inline', defeating the
           purpose of Content-Security-Policy.
  Fix:     Replaced with nonce-based style-src: 'self' 'nonce-{nonce}'.

[FIXED] C-03: XSS via esc() Function
  File:    web/static/js/components.js
  Issue:   esc() escape function did not escape single quotes (&#x27;)
           or backticks (&#x60;), allowing XSS in attribute contexts.
  Fix:     Added single-quote and backtick escaping.

[FIXED] C-04: XSS via Toast innerHTML
  File:    web/static/js/components.js
  Issue:   Toast show() used innerHTML, enabling script injection
           through user-controlled toast messages.
  Fix:     Replaced with safe DOM API (createElement + textContent).

[FIXED] C-05: Dead CSRF Placeholder
  File:    web/routes/pages.py
  Issue:   _csrf_token() was a dead placeholder returning empty string.
           get_csrf_token never actually called.
  Fix:     Imported real get_csrf_token from web.middleware.csrf.

[FIXED] C-06: Logo Upload Path Traversal
  File:    web/routes/gym_profile.py
  Issue:   Old logo deletion did not validate the resolved path was
           within the upload directory. Attacker could craft filename
           to delete arbitrary files.
  Fix:     Added is_relative_to() check before deletion.

[FIXED] C-07: Logo Upload MIME Validation
  File:    web/routes/gym_profile.py
  Issue:   No magic-byte validation on uploaded logos. Only checked
           file extension.
  Fix:     Added magic byte verification (PNG \x89PNG, JPEG \xff\xd8\xff,
           WebP RIFF).

[FIXED] C-08: AuthAuditLog Schema Mismatch
  File:    web/database/models.py
  Issue:   AuthAuditLog.id was Integer but service layer generated UUID
           strings, causing type mismatch errors.
  Fix:     Changed to String(36) with uuid4 default.

[FIXED] C-09: Duplicate Import in Auth
  File:    web/routes/auth.py
  Issue:   log_auth_event_from_request and AuthEventType imported twice
           (module-level and inside function).
  Fix:     Removed redundant inner import.

[FIXED] C-10: GymProfile Color Defaults Mismatch
  File:    web/database/repository.py
  Issue:   Repository _gym_profile_to_dict() had color defaults
           (#FFE500/#2DD4BF) that didn't match GymProfile model
           defaults (#E5B800/#292524).
  Fix:     Aligned repository defaults to match model.

[FIXED] C-11: Missing Updatable Fields in Gym Profile
  File:    web/database/repository.py
  Issue:   actualizar_gym_profile() didn't update razon_social,
           regimen_fiscal, codigo_postal_fiscal, uso_cfdi,
           horarios_comidas fields.
  Fix:     Added missing fields to update loop.

[FIXED] C-12: Fragile File Path for Branding
  File:    web/routes/pages.py
  Issue:   Used os.path.join() with fragile relative path for branding.json.
  Fix:     Replaced with Path() object using __file__ as anchor.

SECTION 3 — FIXES APPLIED IN PHASE 4 (QA DEBUGGER)
════════════════════════════════════════════════════════════════════════════════

[FIXED] F-01: Billing Test Assertion Too Strict
  File:    tests/test_billing_consolidation.py
  Issue:   test_pagos_no_inline_stripe_session_create failed because it
           banned ALL references to stripe.checkout.Session.create, even
           inside @retry_with_backoff wrapper functions.
  Fix:     Relaxed assertion to only fail if route handlers call Stripe
           directly (allows wrapped utility functions).

[FIXED] F-02: pagos.py Missing subscription_service Delegation
  File:    api/routes/pagos.py
  Issue:   pagos.py did not import or use activate_subscription,
           complete_checkout, or is_payment_processed from
           web.services.subscription_service.
  Fix:     Added import; wired MP webhook to use shared services with
           idempotency check instead of TODO placeholder.

[FIXED] F-03: Stripe Price ID Aliases Missing
  File:    config/settings.py
  Issue:   Settings had STRIPE_PRICE_STARTER, STRIPE_PRICE_PROFESIONAL,
           STRIPE_PRICE_CLINICA but tests expected English aliases
           STRIPE_PRICE_PRO and STRIPE_PRICE_ENTERPRISE.
  Fix:     Added alias properties.

[FIXED] F-04: GymSettings Model Missing
  File:    web/database/models.py
  Issue:   GymSettings SQLAlchemy model never defined. Repository and
           tests imported it, causing ImportError.
  Fix:     Created GymSettings model with id, gym_id, trial_days,
           trial_max_clients, strict_mode, timezone, locale, timestamps.

[FIXED] F-05: GymLicense Model Missing
  File:    web/database/models.py
  Issue:   GymLicense SQLAlchemy model never defined. Repository and
           tests imported it, causing ImportError.
  Fix:     Created GymLicense model with id, gym_id, license_key,
           plan_tier, max_clients, is_active, expires_at, revoked_at,
           payment_provider, payment_reference, timestamps.

[FIXED] F-06: Pagination Limit le=500
  File:    api/routes/clientes.py
  Issue:   Clientes list endpoint allowed up to 500 results per page
           (le=500), enabling DoS via large queries.
  Fix:     Changed to le=100.

[FIXED] F-07: CSRF Blocking API Routes
  File:    api/app.py
  Issue:   CSRF middleware only exempted /api/billing/webhook,
           /api/billing/mp/webhook, and /api/auth/. All other API
           endpoints (using Bearer tokens) were blocked by CSRF
           when no token header present, returning 403 instead of 401.
  Fix:     Exempted all /api/ paths from CSRF (they use Bearer auth).

[FIXED] F-08: CSRF Meta Tag Missing on Index
  Files:   web/routes/pages.py, web/templates/index.html
  Issue:   Homepage (/) did not include csrf-token meta tag because
           index.html doesn't extend base.html.
  Fix:     Added csrf_token to template context and meta tag to index.html.

SECTION 4 — REMAINING ISSUES (NOT FIXED — REQUIRE PLANNING)
════════════════════════════════════════════════════════════════════════════════

HIGH PRIORITY:

  H-01: No Redis Rate Limiting
    File:    web/rate_limiter.py
    Issue:   In-memory rate limiting doesn't work across multiple pods/servers.
    Impact:  Any multi-server deployment has no effective rate limiting.
    Fix:     Implement Redis-backed rate limiter for production.

  H-02: plans.py Global CATEGORIAS Race Condition
    File:    core/generador_planes.py / web/routes/plans.py
    Issue:   Some plan generation code modifies global CATEGORIAS dict,
             causing race conditions under concurrent requests.
    Fix:     Make copies of CATEGORIAS before modification.

  H-03: Email PDF Path Traversal
    File:    api/factura_pdf.py
    Issue:   PDF generation may accept user-supplied paths without
             proper validation.
    Fix:     Validate all file paths are within allowed directories.

  H-04: MP Webhook Signature Verification
    File:    api/routes/pagos.py
    Issue:   MercadoPago webhook does not verify request signatures,
             allowing anyone to send fake webhook notifications.
    Fix:     Implement HMAC signature verification similar to Stripe.

  H-05: Feature Flags Don't Hard-Fail in Production
    File:    config/feature_flags.py
    Issue:   Feature flags silently default when misconfigured in prod.
    Fix:     Add strict mode for production that raises on unknown flags.

  H-06: Alembic Migration Check at Startup
    File:    web/main_web.py
    Issue:   No verification that DB schema matches expected state.
             create_all() in dev can mask migration gaps.
    Fix:     Add startup check comparing alembic head vs current.

  H-07: 20+ "C1 FIX" Comments Indicate Incomplete RBAC Migration
    Files:   Multiple route files
    Issue:   Multiple TODO/FIXME comments marked "C1 FIX" suggest
             RBAC system rollout was not fully completed.
    Fix:     Complete RBAC checks in all route handlers.
